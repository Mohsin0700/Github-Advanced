Git & Github

  • Git is a tool which is used for version control, a version control tool is used to track changes in a project.
  • Github is website which allows us to store our code and manage code history using git.

Git Commands

  1. Git Init

    This commands initializes a git repository in the current directory and it is necessary to run this command first before doing any git operations.

  2. Git Clone "HTTPS Link of Github Repository"

    This command is used to clone a repository from github

  3. Git Status"

    This command is used to check the status of the git repository.

    Note: Whenever we use this command it shows 4 type of options/statuses

    • Untracked files

      This is a file which is not tracked by git

    • Modified / Unstaged Files

      This is a file which is not staged. Which means file is modified but has not been added/staged to git

    • Unmodified Files

      An old committed file which is not changed.

    • Staged Files

      Whenever we add a file either if it is a new file or modified file it is staged
