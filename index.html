Github Advance

Git & Github

  • Git is a tool which is used for version control, a version control tool is used to track changes in a project.
  • Github is a website which allows us to store our code and manage code history using git.

Git Commands

  1. Git Init

    This commands initializes a git repository in the current directory and it is necessary to run this command first before doing any git operations.

  2. Git Clone "HTTPS Link of Github Repository"

    This command is used to clone a repository from github

  3. Git Status"

    This command is used to check the status of the git repository.

    Note: Whenever we use this command it shows 4 type of options/statuses

    • Untracked files

      This is a file which is not tracked by git

    • Modified / Unstaged Files

      This is a file which is not staged. Which means file is modified but has not been added/staged to git

    • Unmodified Files

      An old committed file which is not changed.

    • Staged Files

      Whenever we add a file either if it is a new file or modified file it is staged

  4. Git Add "File Name" OR Git Add .

    This command is used to add a file to git repository.If we write git add . it will add all the files in the directory otherwise we can add an individual file by writing git add "File Name". After this command we can see the file in the staging area

  5. Git Commit -m "Message"

    This command is used to commit a file to git repository. Which means it will pickup the files from staging area and add it to the commit history which can be seen in git log and github.

  6. Git remote add origin "HTTPS Link of Github Repository"

    This command is necessary to run once to connect the local repository to github, after this command we can push the code to github

  7. Git Push

    This command is used to push the code to github.

  8. Git remote -v

    This command is used to view the address of connected remote repository.

Branches in Git

Definition:

Whenever two or more developers work on the same project they can create multiple branches. Creating branch means everyone is creating their own copy of the project of that poin from where branch is created.

  1. git branch

    This command is used to see all available branches

  2. git branch -M "Branch Name"

    This command is used to change the name of the current branch

  3. git checkout -b "Branch Name"

    This command is used to create a new branch, when we create a new branch by default we navigate to that branch

    Note: When we create a new branch it means we are creating a copy of the project from the that point when branch is created

  4. git checkout "Branch Name"

    This command is used to navigate to a particular branch

  5. git branch -d "Branch Name"

    This command is used to delete the branch locally and keep working tree clean. We often use this branch once the feature on which we were working is merged to the main branch. It helps to clean the working tree

    Note: We can't delete the main branch and to delete a branch first we must move from that branch

  6. git push origin --delete "Branch Name"

    This command is used to delete the branch from github
